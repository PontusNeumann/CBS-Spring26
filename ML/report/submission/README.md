# Submission bundle

Code, data, and report assets that accompany the written report for the KAN-CDSCO2004U Machine Learning and Deep Learning final project.

## What is in here

```
submission/
├── README.md                   <- this file
├── requirements.txt            <- pinned Python dependencies
├── data/
│   └── consolidated_modeling_data.parquet   <- the only data file you need
│       + README.md, MISSING_DATA.md, release-manifest, info.json
├── scripts/                    <- six teacher-friendly scripts + the EDA script
│   ├── config.py               single source of paths and seeds
│   ├── 01_data_prep.py         load + leakage checks + feature list
│   ├── 02_features.py          feature taxonomy + scaler + Isolation Forest score
│   ├── 03_train_models.py      train 7-8 models with GroupKFold CV + complexity benchmark
│   ├── 04_calibration.py       isotonic calibration + reliability + bootstrap CI + permutation importance
│   ├── 05_backtest.py          realistic capital-aware backtest + naive baseline + headline figure
│   └── 06_tuning_optuna.py     Optuna TPE tuning for RF, HistGBM, MLP
├── outputs/                    <- created on first run; teacher can delete and regenerate
│   ├── data/                   leakage report, feature list, scaler, IsoForest scores
│   ├── models/<model>/         OOF + raw test + calibrated test predictions, metrics.json
│   ├── metrics/                comparison table, calibration summary, AUC CIs, permutation importance
│   ├── backtest/               sensitivity grid, falsification verdict, overview heatmap
│   └── tuning/<model>/         Optuna study history, best params, tuned test predictions
└── report_assets/              <- frozen artefacts cited in the .docx report
    ├── figures/
    │   ├── main/               headline figures used in the report body
    │   └── appendix/eda/       19 EDA figures + tables + the EDA appendix .docx
    └── tables/                 tuning summary, backtest sensitivity, tuned hyperparameters
```

## How to run

```bash
# 1. set up a Python 3.11+ environment, then:
pip install -r requirements.txt

# 2. run the pipeline in order; every script is self-contained and idempotent
cd scripts/
python 01_data_prep.py            # ~30 s
python 02_features.py             # ~5 min  (Isolation Forest fit on 1.1M rows)
python 03_train_models.py         # ~60 min (CV + complexity benchmark on 7-8 models)
python 04_calibration.py          # ~10 min (bootstrap CI + permutation importance)
python 05_backtest.py             # ~5 min  (capital-aware sim across 540 cells)
python 06_tuning_optuna.py        # ~120 min (RF + HistGBM × 30 trials)
```

If you only want the headline numbers, run scripts 01-05 (skip 06). Tuning is optional and the report uses the side-by-side baseline-vs-tuned figure already in `report_assets/figures/main/`.

EDA is generated by `../../report_tools/eda.py` (kept outside the submission since it is still work-in-progress and writes directly into `report_assets/figures/appendix/eda/`).

## Data

The single source of truth is `data/consolidated_modeling_data.parquet` (303 MB):

- 1,371,180 rows total (1,114,003 train + 257,177 test)
- 87 columns: 5 meta/id, 70 core features, 12 wallet features
- Target: `bet_correct` (binary, ~50.3% positive in both splits)
- Train/test split is by market cohort (63 train markets, 10 test markets, zero overlap) — not random — so no market spans both splits
- Build date: 2026-04-29 (see `data/release-manifest-2026-04-29.md` for SHA-256)

The 12 wallet features are already joined into the parquet. There is no need to load `wallet_enrichment.parquet` separately.

Missing-value policy is documented in `data/MISSING_DATA.md`.
